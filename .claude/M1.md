# M1 — Guest Program + Dev Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use `- [ ]` checkboxes.
> Executor = Claude Code dgn konteks project ini + skill Stellar/Soroban. Boilerplate `cargo risczero new` (skeleton `creator-badge`, sudah hijau di M0) TIDAK diulang di sini — hanya delta M1.

**Goal:** Guest RISC Zero meng-commit journal bersih `{eligible, commitment, nullifier, target}` dari `secret` privat + `target_contract` publik, dijalankan host di dev mode, tervalidasi tanpa proving asli.

**Architecture:** Logika hashing diekstrak ke crate `shared` (no_std + sha2) yang dipakai guest DAN test host, sehingga layout byte journal punya satu sumber kebenaran. Guest menulis journal via `commit_slice` (raw bytes, bukan serde) supaya `journal_raw` deterministik & gampang di-hash kontrak di M2.

**Tech Stack:** risc0-zkvm 3.0.5 (dev mode), sha2 0.10, Rust host 1.96.1 / guest 1.94.1.

## Global Constraints
- SHA-256 di guest untuk SEMUA hash. Tiga hash beda input, jangan tertukar:
  `commitment=sha256(secret)`, `nullifier=sha256(secret||target)`, `journal_hash=sha256(journal_raw)` (yg terakhir baru dipakai M2).
- Secret TIDAK pernah masuk journal.
- Receipt Groth16 wajib untuk on-chain (M5) — di M1 dev mode saja.
- **Journal layout terkunci (97 byte, fixed, LSB `eligible` di depan):**
  ```
  [0]        eligible: u8 (1=true)
  [1..33]    commitment [u8;32]
  [33..65]   nullifier  [u8;32]
  [65..97]   target_contract [u8;32]
  ```
  Ini adalah interface yang M2 (kontrak) akan parse. Jangan pakai risc0-serde untuk journal — pakai `commit_slice`.
- Urutan input guest (`env::read`): `secret: [u8;32]` dulu, lalu `target_contract: [u8;32]`.

- [ ] **Step 1: Failing test** — vektor independen (hitung sha256 manual via sha2 di test), assert tiap slice journal.

```rust
#[test]
fn journal_layout_and_hashes() {
    let secret = [7u8; 32];
    let target = [9u8; 32];
    let j = build_journal(&secret, &target);
    assert_eq!(j.len(), JOURNAL_LEN);
    assert_eq!(j[0], 1);
    assert_eq!(&j[1..33],  &sha256(&secret)[..]);
    let mut cat = secret.to_vec(); cat.extend_from_slice(&target);
    assert_eq!(&j[33..65], &sha256(&cat)[..]);
    assert_eq!(&j[65..97], &target[..]);
    // secret tak bocor
    assert!(!j.windows(32).any(|w| w == secret));
}
```

- [ ] **Step 2: Run fail** — `cargo test -p shared` → FAIL (fn belum ada).

- [ ] **Step 3: Implement minimal**

```rust
#![no_std]
use sha2::{Sha256, Digest};
pub const JOURNAL_LEN: usize = 97;

pub fn sha256(input: &[u8]) -> [u8; 32] {
    let mut h = Sha256::new(); h.update(input); h.finalize().into()
}

pub fn build_journal(secret: &[u8; 32], target: &[u8; 32]) -> [u8; JOURNAL_LEN] {
    let commitment = sha256(secret);
    let mut cat = [0u8; 64];
    cat[..32].copy_from_slice(secret);
    cat[32..].copy_from_slice(target);
    let nullifier = sha256(&cat);
    let mut j = [0u8; JOURNAL_LEN];
    j[0] = 1;
    j[1..33].copy_from_slice(&commitment);
    j[33..65].copy_from_slice(&nullifier);
    j[65..97].copy_from_slice(target);
    j
}
```
> Test butuh std → tambahkan di `shared/Cargo.toml`: `[features] std=["sha2/std"]` dan `[dev-dependencies] sha2={version="0.10"}` ATAU cukup jalankan test dgn `sha2` default host. Paling simpel: buat crate `no_std` tapi test host otomatis link std — `sha2` no_std sudah cukup, hapus `#![no_std]` guard di test module bila perlu.

- [ ] **Step 4: Run pass** — `cargo test -p shared` → PASS.

use methods::{CREATOR_BADGE_GUEST_ELF as ELF}; // sesuaikan nama const
use shared::build_journal;

#[test]
fn dev_mode_journal_matches() {
    std::env::set_var("RISC0_DEV_MODE", "1");
    let secret = [7u8; 32];
    let target = [9u8; 32];
    let env = ExecutorEnv::builder()

        .write(&target).unwrap()
